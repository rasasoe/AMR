# Contributing

Thank you for your interest in contributing to this project! A quick guide:

- Bug reports: open an issue describing the problem and steps to reproduce.
- Feature requests: open an issue with motivation and suggested API changes.
- Pull requests: fork, create a feature branch, add tests if applicable, and open a PR.

Style:
- Use clear, concise commit messages.
- Keep code readable and add comments for non-obvious logic.

Development tips:
- Add images to `assets/images/` and reference them from `README.md`.
- For video demos, add MP4 to `assets/video/` and reference the filename in the README.
